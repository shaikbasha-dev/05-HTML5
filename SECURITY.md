# Security Policy

## Overview

The **05-HTML5** repository is an educational, open-source project designed to help learners understand HTML5 through structured documentation, practical examples, and hands-on exercises.

Although this repository does not process user data, run backend services, or store sensitive information, maintaining a secure and trustworthy learning environment remains a priority.

| Version | Supported |
|----------|:---------:|
| v1.x.x | ✅ Yes |
| Earlier Versions | ❌ No |

---

# Reporting a Security Vulnerability

If you discover a potential security issue related to this repository, please report it responsibly.

Examples include:

- Malicious or unsafe code
- Harmful scripts
- Security-related documentation errors
- Repository misconfigurations
- Supply chain or dependency concerns (if applicable in future)

Please include the following information:

- A clear description of the issue
- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Screenshots or supporting evidence (if applicable)
- Suggested solution (optional)

---

# Response Process

After receiving a valid security report, the repository maintainer will:

1. Review the report.
2. Verify the reported issue.
3. Assess the potential impact.
4. Prepare an appropriate fix.
5. Update the repository if necessary.
6. Acknowledge the contribution where appropriate.

---

# Responsible Disclosure

Please do not publicly disclose a security vulnerability until it has been reviewed and, if necessary, addressed.

Responsible disclosure helps protect users and maintain the integrity of the repository.

---

# Security Best Practices

When using this repository:

- Download the repository only from the official GitHub source.
- Review code before using it in production projects.
- Keep your local development environment updated.
- Follow HTML5 best practices.
- Validate external resources before integrating them into your own projects.
