# Key Concepts

## HTML Element

An HTML element consists of:

- Opening tag
- Content
- Closing tag

Example:

```html
<p>Welcome to HTML5</p>
```

---

## HTML Tag

Tags define the beginning and end of HTML elements.

Example:

```html
<h1>
</h1>
```

---

## Opening and Closing Tags

- Opening tag starts an element.
- Closing tag ends an element.
- Most HTML elements require both tags.

Example:

```html
<p>Hello World</p>
```

---

## Empty (Void) Elements

These elements:

- Do not contain content.
- Do not require closing tags.

Examples:

```html
<br>

<hr>

<img>

<input>
```

---

## Block-Level Elements

Characteristics:

- Begin on a new line.
- Occupy the full available width.
- Organize webpage structure.

Examples:

- `<div>`
- `<section>`
- `<article>`
- `<header>`
- `<footer>`

---

## Inline Elements

Characteristics:

- Remain on the same line.
- Occupy only the required width.
- Used for formatting small portions of content.

Examples:

- `<span>`
- `<a>`
- `<strong>`
- `<em>`
- `<img>`

---

## Nesting

Proper nesting creates a parent-child relationship between HTML elements.

Correct Example:

```html
<div>

    <p>HTML5 Tutorial</p>

</div>
```

Proper nesting improves readability, maintainability, accessibility, and browser compatibility.

---

## HTML Attributes

Attributes provide additional information about HTML elements.

Example:

```html
<a href="https://www.example.com">

    Visit Website

</a>
```

Common attributes:

- `id`
- `class`
- `href`
- `src`
- `alt`
- `title`

---

## Global Attributes

Global attributes can be applied to most HTML elements.

Examples:

- `id`
- `class`
- `style`
- `title`
- `lang`
- `hidden`
- `tabindex`
- `contenteditable`

---

## HTML Best Practices

Professional HTML code should:

- Use semantic elements.
- Maintain proper nesting.
- Use meaningful attribute values.
- Follow consistent indentation.
- Use lowercase element names.
- Use double quotation marks for attribute values.
- Write accessible HTML.
- Keep code clean and organized.

# Quick Comparison

| Concept | Description |
|----------|-------------|
| HTML Element | Opening tag + Content + Closing tag |
| HTML Tag | Markup enclosed within angle brackets |
| Empty Element | No content and no closing tag |
| Block-Level Element | Starts on a new line and occupies full width |
| Inline Element | Occupies only required width and stays on the same line |
| HTML Attribute | Additional information about an element |
| Global Attribute | Can be used with most HTML elements |
| Nesting | Parent-child relationship between elements |

---

# Best Practices Checklist

Before completing an HTML document, ensure that:

- HTML5 document structure is correct.
- Elements are properly nested.
- Required closing tags are present.
- Empty elements are used correctly.
- Semantic HTML elements are used whenever appropriate.
- Attribute values are meaningful.
- Images include descriptive `alt` text.
- Duplicate `id` values are avoided.
- Code is properly indented.
- HTML validates without errors.
