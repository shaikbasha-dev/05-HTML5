# Headings and Paragraphs Summary

## Module Overview

Headings and paragraphs are the foundation of content organization in HTML. They provide structure, improve readability, enhance accessibility, and help search engines understand the hierarchy of information on a webpage.

This module covered HTML heading elements, heading hierarchy, paragraphs, line breaks, horizontal rules, whitespace handling, formatting techniques, and industry best practices for writing clean and semantic HTML5 documents.

# Important HTML Elements

| Element | Purpose |
|----------|---------|
| `<h1>` | Main heading |
| `<h2>` | Major section heading |
| `<h3>` | Subsection heading |
| `<h4>` | Lower-level heading |
| `<h5>` | Minor heading |
| `<h6>` | Lowest-level heading |
| `<p>` | Paragraph |
| `<br>` | Line break |
| `<hr>` | Thematic break |
| `<pre>` | Preserves whitespace |

# Best Practices Checklist

Always remember to:

- Use one primary `<h1>` for the page title.
- Follow a logical heading hierarchy.
- Write meaningful and descriptive headings.
- Keep paragraphs concise and focused.
- Use paragraphs instead of multiple `<br>` elements.
- Use `<hr>` only for thematic separation.
- Preserve formatting with `<pre>` when required.
- Use `&nbsp;` only when necessary.
- Maintain proper indentation.
- Write semantic HTML.
- Improve accessibility.
- Follow SEO-friendly practices.

---

# Common Mistakes to Avoid

Avoid the following mistakes:

- Using multiple `<h1>` elements unnecessarily.
- Skipping heading levels.
- Using headings only for visual styling.
- Writing long, difficult-to-read paragraphs.
- Using multiple `<br>` elements for spacing.
- Using `<hr>` only as decoration.
- Expecting multiple spaces to appear in browser output.
- Overusing `&nbsp;` for page layout.
- Ignoring semantic HTML principles.
